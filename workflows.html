Common PVM Workflows

Step-by-step guides for the most common tasks you will do with PVM. For a full command listing, run pvm -h. For contextual help, run pvm help.

New Project

Starting a new Perl project from scratch takes four steps. PVM sets up dependency management and development tooling for you.

  1. Initialize the project:

    pvm workspace init my-app
cd my-app

  2. Add your dependencies:

    pvm module add DBI
pvm module add Test::More --dev

  3. Install dependencies:

    pvm module install

  4. Start development:

    pvm dev

Existing Project

When you clone or inherit a project that already uses PVM, get up and running in four steps. The dev command provides file watching, automatic builds, and test running.

  1. Check the workspace status:

    pvm workspace status

  2. Install dependencies:

    pvm module install

  3. Run the test suite:

    pvm test

  4. Start development mode:

    pvm dev

Module Development

Developing a CPAN module requires a few extra steps compared to a plain application. PVM's module template wires up the metadata you need for distribution. The build command creates a CPAN-ready distribution in the build/ directory.

  1. Initialize with the module template:

    pvm workspace init --template=module My::Module

  2. Set up distribution metadata by editing pvm.toml to configure your author name, license, and other distribution fields.

  3. Start development:

    pvm dev

  4. Build for distribution:

    pvm build

  5. Test the distribution:

    cd build && perl Makefile.PL && make test

Testing

PVM automatically discovers tests in the t/ directory. You can run all tests, target a specific file, collect coverage, or keep tests running continuously while you work.

  1. Run all tests:

    pvm test

  2. Run a specific test file:

    pvm test t/basic.t

  3. Run tests with coverage:

    pvm test --coverage

  4. Continuous testing — use pvm dev, which includes automatic test running on file changes:

    pvm dev

  5. Debug test failures with verbose output:

    pvm test --verbose

Building

PVM supports several build modes depending on where you are in your development cycle. All build outputs go to the build/ directory and include the CPAN metadata your distribution needs.

  1. Development build — fast, generates .pmc files inline:

    pvm build --inline

  2. Distribution build for CPAN:

    pvm build

  3. Continuous build — rebuilds automatically when files change:

    pvm build --watch

  4. Clean build — remove previous output and rebuild from scratch:

    pvm build --clean
